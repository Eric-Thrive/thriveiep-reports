Import
  • organization: Matched case-insensitively against existing orgs. Also accepts organisation. Row fails if not found.
  • coach_email: Must match a user with the Coach role. Row fails if not found.
  • Per-row atomicity: Each row is wrapped in a DB transaction. One row's failure doesn't affect others.
  • Students are created dormant (no PIN, no email). Go to the Roster to send invites.

Troubleshooting
Common problems and solutions for the invite and batch import workflows.
ProblemCauseSolution
"Send Invite" stays disabledMissing required field or no senderFill all fields and select a sender
"Email already exists"Another user has this emailUse "Add Existing User" tab instead
"Username already exists"Duplicate usernameChoose a different username
No senders in dropdownNo @thriveiep.com adminsContact developer to verify admin accounts
User can't log inEmail in spam folderCheck spam. Admin can reset PIN.
"Organization not found"New CSV org name doesn't match Check spelling. Case-insensitive but exact match. Create org first if needed.
"Coach not found"New coach_email not a Coach-role user Verify email and Coach role assignment.
"Template name exists"New Duplicate template name Choose a different name or delete the existing template.

Send invites Student Roster → Select students → Send Invites
