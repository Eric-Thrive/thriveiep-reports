Do students actually engage with results and gain insight?

Q4: What patterns emerge across 50 students for Phase 3?

Usage patterns, result patterns, anything worth testing at scale in the 750-student expansion.

Served by: WS4 dashboard (all views), rollout decision framework, CSV export

Section 4 of 16

Problem Statement

ThriveIEP is running a 50-student design partner pilot with NLU. The SOW (Statement of Work) requires a findings memo with usage data, insights, and a rollout recommendation. Elizabeth's evidence protocol defines what the case study needs. This PRD defines what the software must capture to produce that evidence.

C2A currently records whether students complete assessments and their scores. It does not capture:

  • How students experience the assessment
  • Whether they read their reports
  • Whether advisors access or use the results
  • How either group feels about the process

The findings memo cannot be written without this data, and it cannot be reconstructed after the fact from memory.

Section 5 of 16

Goals

G1: Prove C2A changes student self-understanding and advisor behavior

Generate credible evidence that students engage with results and advisors incorporate C2A into coaching. Core claim of the case study.

G2: Understand how students actually experience C2A

Behavioral signals (linger time, save-exit patterns, report engagement) that reveal where students struggle, disengage, or find value.

G3: Close the advisor feedback loop

Instrument whether advisors access reports and which sections they review. Without this, the assessment-to-coaching pipeline is a black box.

G4: Collect structured feedback from both students and advisors

The SOW explicitly requires feedback from both groups. Effort scores, check-ins, open-text prompts must be capturable in the software.

G5: Build reusable analytics infrastructure

The telemetry layer should outlive the NLU pilot. Every future deployment needs the same usage data. Build once, configure per-deployment.

G6: Capture minimum coaching integration evidence New v4

Even with the full coaching session log deferred, capture two binary data points: (a) did a coaching session occur? (b) was C2A explicitly discussed? These directly serve Elizabeth's Q2 and Q3.

Requirements: WS1-3 (Telemetry)

WS1: Analytics Events Layer P0

  • c2aAnalyticsEvents table: id, userId, eventType, eventData (jsonb), pageUrl, sessionId, deploymentId, createdAt
  • Client-side useAnalytics() hook that batches events, fires on page unload
  • POST /api/c2a/analytics/events batch insert, fire-and-forget
  • Event types: item_displayed, item_responded, save_exit, sprint_completed, page_viewed, accordion_opened, report_section_viewed, advisor_report_viewed, feedback_submitted
  • deploymentId for multi-deployment reuse (G5)

Acceptance: Events in DB within 5 seconds. No UX impact (async, non-blocking).

WS2: Report Engagement Tracking P0

  • usePageTracker hook on all 5 Start Here Report pages
  • Track: entry/exit timestamps, dwell time, scroll depth (25/50/75/100%), accordion open/close
  • Works for both student and advisor views (differentiated by userId role)

WS3: Assessment Behavior Tracking P0

  • itemDisplayedAt timestamp when each item renders
  • dwellTimeMs = response time minus display time, persisted with item response
  • Save-exit events with item index, sprint, elapsed time
  • Sprint-level: total duration, pause count, items per minute

Acceptance: "Completion topology" showing exact item where each student saved/exited or lingered.

Section 14 of 16

Rollout Decision Framework & Metrics

Recommend Expansion (ALL must be true)

  • Completion rate ≥ 80% (40/50 students)
  • Report view rate ≥ 70% of completers (3+ of 5 pages)
  • Advisor report access rate ≥ 60%
  • Student effort score median ≤ 4 (if WS8 activated)
  • Advisor utility rating majority positive

Recommend with Modifications (ANY triggers)

  • Completion rate 60-79%
  • Report view rate 50-69%
  • Effort scores median > 4
  • Advisors not accessing reports

Do Not Recommend Yet (ANY triggers)

  • Completion rate < 60%
  • Advisor report access < 30%
  • Effort scores consistently 6-7

Lagging Indicators (end of pilot)

  • Pre/post articulation change: measurable increase in specificity
  • Student effort score: median ≤ 4 on CES
  • Advisor utility rating: majority positive
  • Changed v4 Coaching session data: WS8 advisor check-in (Y/N fields) + NLU systems for detail
  • New v4 "Wouldn't have known" signals: count and themes from WS8 + coach view

Why reorder? Coaches will assume student results are visible as soon as students complete the assessment. If coach view ships after telemetry and admin views, we lose coach trust before collecting any evidence. Coach view MVP uses existing data, not the new analytics layer.
