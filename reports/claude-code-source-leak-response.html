Section 1 of 12

Claude Code Source Leak: What You Need to Know Right Now

Key Messages

  • Your data is safe, but your toolchain needs attention. Anthropic's Claude Code accidentally shipped its full source code in an npm package. No API keys, model weights, or user data were exposed. But a separate, unrelated supply chain attack on the axios npm package did deploy malware on the same day.
  • Check for axios compromise first. The axios attack (versions 1.14.1 and 0.30.4) installed a Remote Access Trojan on affected machines. This is the more urgent threat.
  • Switch to the native Claude Code installer. Anthropic now recommends bypassing npm entirely for Claude Code updates. This eliminates exposure to the npm supply chain.

What happened?

On March 31, 2026, Anthropic published Claude Code version 2.1.88 to npm with a "source map" file accidentally included. A source map is a debugging artifact that maps compressed code back to the original, readable source. It exposed approximately 512,000 lines of TypeScript across 1,900 files. Security researcher Chaofan Shou spotted it and posted publicly. Within hours, the code was archived on GitHub.

Is this actually dangerous?

For most developers: no immediate danger. The leak exposed how Claude Code works internally, not your conversations, files, or credentials. Think of it like someone publishing the blueprint for a bank vault door. The vault itself (your data, API keys) was never opened.

The medium-term risk is that attackers can now study Claude Code's exact security logic and potentially craft more targeted attacks against users who open malicious repositories.

Has this happened before?

Yes. This is Anthropic's second identical mistake. The same source map exposure occurred in February 2025. A separate CMS misconfiguration on March 26 leaked internal documents. Three configuration-level exposures in thirteen months raises fair questions about release processes for a tool trusted with filesystem and terminal access.

Section 2 of 12

What Happened: The Full Picture

On March 31, 2026, version 2.1.88 of @anthropic-ai/claude-code shipped to npm with a 59.8 MB source map file (cli.js.map). This file allowed anyone to reconstruct Anthropic's complete TypeScript codebase: approximately 1,900 files and 512,000 lines of code.

The Root Cause

Claude Code uses Bun as its JavaScript bundler. Bun generates source maps by default. A missing *.map entry in the project's .npmignore file meant the source map was included in the published package. The map file pointed to a zip archive hosted on Anthropic's Cloudflare R2 storage bucket, which anyone could download.

"A single misconfigured .npmignore or files field in package.json can expose everything." — Gabriel Anhaia, dev.to analysis

Anthropic's Response

"Earlier today, a Claude Code release included some internal source code. No sensitive customer data or credentials were involved or exposed. This was a release packaging issue caused by human error, not a security breach. We're rolling out measures to prevent this from happening again." — Anthropic spokesperson, Fortune, 2026-03-31

The Same-Day Coincidence

Completely unrelated: the axios npm package (100M+ weekly downloads) was also compromised on March 31 by a North Korean threat actor. This was an actual malicious attack, not an accident. Two npm supply chain disasters in one day.

What Was Exposed

HIGH INTEREST System Prompts & Tool Definitions

The full instruction set that governs how Claude Code behaves: how it asks for permission, which tools it can use, and what safety guardrails it follows. This is the "agentic harness" that sits around the AI model itself.

HIGH INTEREST 44 Unreleased Feature Flags

Fully built but not yet shipped capabilities, sitting behind compile-time flags:

  • Background agents running 24/7 with GitHub webhooks and push notifications
  • One Claude orchestrating multiple worker Claudes, each with restricted tools
  • Cron scheduling for agents (create, delete, list jobs)
  • Full voice command mode with its own CLI entrypoint
  • Browser control via Playwright (real browser, not just web fetch)
  • Agents that can sleep and self-resume without user prompts
  • Persistent memory across sessions without external storage

NOTABLE Internal Model Codenames

Capybara (Claude 4.6 variant), Fennec (Opus 4.6), Numbat (prelaunch Haiku), Tengu (experimental). These confirm Anthropic's upcoming model tiers and naming conventions.

SECURITY Bash Validator Logic & Bypass Env Vars

bashSecurity.ts: 2,500+ lines of bash command validation. Plus two environment variables that disable safety features entirely: CLAUDE_CODE_ABLATION_BASELINE (disables all safety) and DISABLE_COMMAND_INJECTION_CHECK (skips injection guard). Presumably for internal testing, but now public knowledge.

CONTROVERSIAL "Undercover Mode"

A subsystem that tells Claude Code to hide all evidence of AI authorship when Anthropic employees contribute to public open-source repositories. The system prompt says: "Do not blow your cover." Heated debate on Hacker News about the ethics of undisclosed AI contributions.

What Was NOT Leaked

Safe

  • Model weights and training data
  • API keys, tokens, or customer credentials
  • User conversation data
  • Server-side infrastructure code

Exposed

  • Client-side agentic harness code
  • System prompts and tool definitions
  • Security validation logic
  • Unreleased feature roadmap

The axios Supply Chain Attack (Same Day, Unrelated)

CRITICAL This is the more urgent threat from March 31.

The axios HTTP library, one of npm's most popular packages with over 100 million weekly downloads, was compromised by threat actor UNC1069 (attributed to North Korea by Google GTIG). The attacker hijacked the primary maintainer's npm account.

What Happened

  • Attacker changed the maintainer's email to a Proton Mail address they controlled
  • Published two malicious versions: axios@1.14.1 and axios@0.30.4
  • Both contained a phantom dependency (plain-crypto-js@4.x) with a postinstall hook
  • The hook downloaded a platform-specific Remote Access Trojan (RAT)
  • Three payloads (Windows, macOS, Linux) were pre-staged 18 hours in advance
  • Attack window: 00:21 to 03:29 UTC on March 31, 2026
  • After installing, the dropper deleted itself and falsified its own version number

Check Your Machine Now

# Check lockfiles for compromised versions grep -E '1\.14\.1|0\.30\.4' package-lock.json yarn.lock bun.lockb 2>/dev/null # Check for phantom dependency (the malware delivery package) ls node_modules/plain-crypto-js 2>/dev/null && echo 'POTENTIALLY COMPROMISED'

If you find either indicator: treat the machine as fully compromised. Isolate it from your network. Rotate all credentials stored on or accessed from that machine. Reimage the operating system.

Immediate Actions (Today)

CRITICAL 1. Check for axios RAT

Run the commands from the previous slide. This is the highest-priority action because it involves active malware, not just source code exposure.

HIGH 2. Pin Claude Code to a Safe Version

Do not run version 2.1.88 (the leaked version).

Option A (preferred): Migrate to the native installer, which bypasses npm entirely:

curl -fsSL https://claude.ai/install.sh | bash

Option B: Downgrade via npm to the last known-safe version:

npm install -g @anthropic-ai/claude-code@2.1.86

HIGH 3. Rotate API Keys

If you ran version 2.1.88 with custom MCP configurations that reference credentials (database connection strings, API tokens, etc.), rotate those credentials now. The leaked source code reveals exactly how Claude Code reads and processes MCP server configurations.

Section 6 of 12

Short-Term Actions (This Week)

MODERATE 4. Audit Your npm Publish Pipeline

If you maintain any npm packages, verify that source maps and debug artifacts are excluded from your published packages:

# Preview exactly what npm will publish npm pack --dry-run

Add a CI/CD pre-publish check that blocks source maps:

set -euo pipefail npm pack >/dev/null PKG="$(ls *.tgz | head -n1)" WORKDIR="$(mktemp -d)" tar -xzf "$PKG" -C "$WORKDIR" if find "$WORKDIR/package" -type f | grep -E '\.map$' >/dev/null; then echo "FAIL: sourcemap found in package" exit 1 fi

MODERATE 5. Harden Your Supply Chain

  • Always commit lockfiles and use npm ci (not npm install) in CI/CD for reproducible builds
  • Consider npm config set min-release-age 3 for 48-72 hour quarantine on new package versions
  • Use --ignore-scripts in CI/CD to block postinstall hooks
  • Evaluate Socket.dev, Snyk, or npm audit for dependency monitoring

MODERATE 6. Review Claude Code Permissions

  • Audit .claude/settings.json for hooks that execute on session start
  • Review MCP server configurations for sensitive credential references
  • The leaked code shows exact hook and MCP orchestration logic, so attackers now know exactly what surfaces to target

Medium-Term Actions (Ongoing)

MODERATE 7. AI Tool Governance Posture

  • Add "third-party AI tool incidents" as a category in your incident response playbook
  • Review AI acceptable use policies against now-known but previously undisclosed features: background agents, remote settings polling, kill switches
  • Monitor Anthropic's security advisories for follow-up patches

MODERATE 8. npm Publishing Best Practices

If you publish npm packages:

  • Use an explicit files whitelist in package.json instead of relying on .npmignore blocklists. Whitelists are safer than blocklists because they fail closed.
  • Configure Bun, webpack, or esbuild to skip source map generation for production builds
  • Add a CI check that rejects packages containing .map files or sourceMappingURL references
"Source maps are source code. Treating them like harmless debug files is a critical vulnerability."
Section 8 of 12

Where the Community Agrees and Disagrees

Strong Consensus

These points are corroborated across Hacker News, Reddit, The Register, Fortune, and dev.to:

  • The source leak is not an emergency for most Claude Code users. No API keys, model weights, or user data were exposed.
  • The concurrent axios attack is potentially dangerous. Check your lockfiles immediately.
  • Migrating from npm to the native installer is the right move going forward.
  • The "Undercover Mode" revelation is ethically concerning but not a security risk.

Contested Points

How serious is the security validator exposure?

One side: Exposing bashSecurity.ts logic and bypass environment variables meaningfully lowers the cost of crafting attacks against Claude Code users.

Other side: Claude Code's security model already assumes the client is untrusted. The server-side model does the real security enforcement.

Does the repeat matter?

One side: Two identical source map exposures (Feb 2025, Mar 2026) plus a CMS leak (Mar 26, 2026) raise legitimate questions about Anthropic's release processes for a tool trusted with deep filesystem access.

Other side: Configuration errors happen at every company. The impact is limited because the code was partially known through prior reverse engineering (CCLeaks).

Competitive blueprint or nothing new?

Fortune: This gives competitors "a literal blueprint for constructing high-agency, commercially viable AI agents."

The Register: Much was already known. This is "a useful comparison point and update source" rather than a revelation.

Section 10 of 12

Gaps and Open Questions

  • No CVE assigned. No formal vulnerability identifier has been issued for the source map leak itself. The prior vulnerabilities (CVE-2025-59536, CVE-2026-21852) were for different issues that have been patched.
  • Are the bypass env vars functional? The exposed code contains CLAUDE_CODE_ABLATION_BASELINE and DISABLE_COMMAND_INJECTION_CHECK, but it is unclear whether these are active in the distributed binary or only in internal builds.
  • New attack surfaces unknown. No analysis exists of whether the exposed orchestration logic creates attack surfaces beyond the already-patched CVEs. The Check Point vulnerabilities showed that hooks and MCP configs could be weaponized. The leaked code makes studying these surfaces cheaper.
  • Competitive impact is speculative. Whether competitors can meaningfully accelerate their products using the leaked code depends on factors we cannot assess from community sources alone.
  • Community sources are inherently unverified. Factual claims are corroborated across multiple sources, but details may shift as deeper analysis emerges over the coming days.

CRITICAL Scan for the axios RAT now

This is the one action that cannot wait. The axios supply chain attack deployed actual malware. Check lockfiles for versions 1.14.1 and 0.30.4, and check for the plain-crypto-js phantom dependency. If present, the machine is compromised.

Evidence: StepSecurity forensic analysis, The Hacker News report, Google GTIG attribution

HIGH Switch Claude Code to the native installer

Anthropic now recommends the native installer over npm. This removes the npm supply chain as an attack surface for your primary development tool. The native installer auto-updates directly from Anthropic's servers.

Evidence: Anthropic's recommendation (Fortune), community consensus across 10+ sources

HIGH Rotate credentials referenced in MCP configs

The leaked code shows exactly how Claude Code reads MCP server configurations and processes credentials. If your .claude/settings.json or MCP configs reference database connection strings, API keys, or other secrets, rotate them. The credentials themselves were not leaked, but the code that processes them was.

Evidence: Check Point CVE-2026-21852 (patched, but demonstrates the attack surface)

MODERATE Add source map checks to your CI pipeline

Anthropic's mistake is a class of error that any team using Bun, webpack, or esbuild can make. A five-line CI check prevents it permanently.

Evidence: Same root cause recurring at Anthropic (Feb 2025, Mar 2026)

Uncertainty: Mostly relevant if you publish npm packages

Open Questions for Further Investigation

  • Are the exposed safety bypass environment variables (CLAUDE_CODE_ABLATION_BASELINE) active in the distributed binary, or do they require internal build flags?
  • Does the exposed hook/MCP orchestration logic create new, unpatched attack surfaces beyond CVE-2025-59536 and CVE-2026-21852?
